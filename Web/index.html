ทำไมต้องเป็น DustWatch?

แก้ปัญหาการวัดฝุ่นที่ไม่สอดคล้องกับสภาพจริง

ค่าฝุ่นจากหน่วยงานรัฐมักวัดจากสถานีขนาดใหญ่ที่มีระยะห่างไกล ซึ่งอาจมีความแตกต่างจากคุณภาพอากาศรอบบริเวณโรงเรียนของคุณอย่างสิ้นเชิง DustWatch เข้ามาปิดช่องว่างนี้

01

โครงข่ายเซนเซอร์ IoT กระจายศูนย์

ติดตั้งเซนเซอร์วัดฝุ่น PM2.5, PM1.0, PM10 และสภาพอากาศแวดล้อม เช่น อุณหภูมิ ความชื้น ความกดอากาศ เสียง และสารเคมีอินทรีย์ระเหยง่าย (TVOCs) ตามจุดสำคัญในโรงเรียน (สนามฟุตบอล, โรงอาหาร, อาคารเรียน)

02

ระบบประมวลผลความน่าเชื่อถือเซนเซอร์ (DCS)

เซนเซอร์วัดฝุ่นระบบแสงเลเซอร์ปกติจะอ่านค่าเพี้ยนสูงขึ้นมากเมื่ออากาศชื้นจัด เช่น หลังฝนตก ระบบของเราใช้อัลกอริทึมวิเคราะห์ความชื้นเพื่อประเมินระดับความน่าเชื่อถือ (Data Confidence Score) เตือนให้ครูระวังผลลัพธ์ที่ผิดเพี้ยน

03

AI Forecasting ทำนายผลล่วงหน้าเพื่อเตรียมพร้อม

ป้อนข้อมูลสภาพอากาศย้อนหลังลงโมเดล Machine Learning เพื่อคาดคะเนทิศทางของฝุ่น ช่วยให้ครูเวรสามารถเลือกกิจกรรมช่วงเช้า/กลางวันให้นักเรียนได้อย่างปลอดภัยสูงสุด

System Architecture

สถาปัตยกรรมและการทำงานร่วมกันของระบบ

DustWatch แบ่งสถาปัตยกรรมออกเป็น 4 ส่วนหลัก ทำงานร่วมกันอย่างไร้รอยต่อเพื่อประสิทธิภาพสูงสุด

ฮาร์ดแวร์เก็บข้อมูลสภาพแวดล้อมภาคสนาม

โหนดฮาร์ดแวร์สร้างขึ้นโดยใช้บอร์ดไมโครคอนโทรลเลอร์ ESP32 เป็นแกนประมวลผลหลัก ร่วมกับเซนเซอร์คุณภาพสูง:

  • PMS5003: เซนเซอร์วัดระดับฝุ่นละอองขนาดเล็ก PM1.0, PM2.5, PM10 ด้วยแสงเลเซอร์
  • BME680: วัดอุณหภูมิ, ความชื้นสัมพัทธ์, ความกดอากาศ และสารระเหยง่าย (Gas TVOCs)
  • เซนเซอร์ระดับความดังเสียง: ตรวจวัดระดับเสียงโดยรอบโรงเรียน
  • การสื่อสาร: ส่งผ่าน Wi-Fi (MQTT protocol) ไปยังฐานข้อมูล โดยรองรับโครงข่าย LoRaWAN สำหรับพิกัดห่างไกล

ฐานการประมวลผล ข้อมูลแบบอนุกรมเวลา และ AI

สมองส่วนกลางของระบบ พัฒนาด้วยความสามารถในการรองรับการรับส่งข้อมูลจำนวนมาก:

  • FastAPI Backend: ประตูเชื่อมรับข้อมูลความเร็วสูงผ่าน REST API และจัดการผู้ใช้
  • InfluxDB Database: ฐานข้อมูล Time-Series เพื่อการบันทึกข้อมูลเซนเซอร์ที่มีการอัปเดตแบบวินาทีต่อวินาที
  • DCS Processing: คำนวณความสมเหตุสมผลของข้อมูลแบบเรียลไทม์เพื่อตัดข้อมูลช่วงมีละอองน้ำ
  • XGBoost & LightGBM: แบบจำลองปัญญาประดิษฐ์ที่ถูกเทรนด้วยข้อมูลสภาพอากาศ เพื่อทำนายผลลัพธ์ระดับฝุ่นในช่วง 1-6 ชม. ข้างหน้า
def calculate_dcs(humidity, temp):
    # คำนวณความชื้นสัมพัทธ์สะสม
    if humidity > 85.0:
        confidence = 100.0 - (humidity - 85.0) * 4.0
    else:
        confidence = 100.0
    return max(10.0, confidence)

แดชบอร์ดแสดงผลเชิงภาพและโต้ตอบได้

ส่วนการนำเสนอข้อมูลเพื่อสร้างความตระหนักรู้และช่วยเหลือในการตัดสินใจ:

  • Interactive School Map: แผนที่จำลองบริเวณโรงเรียนที่ระบุสีของโหนดตามเกณฑ์ดัชนี AQI ล่าสุด
  • Detail Modal: คลิกดูระดับมลพิษทางอากาศ ข้อมูลพยากรณ์ฝุ่น และระดับความน่าเชื่อถือ (DCS)
  • Quick QR Code Public Page: สแกนเพื่อเข้าถึงหน้ารายงานคุณภาพอากาศพกพาแบบด่วนในพิกัดนั้นๆ โดยตรงผ่านสมาร์ทโฟน

หน้าต่างผู้ดูแลระบบและการตั้งค่าเกณฑ์ระบบ

เครื่องมือสำหรับจัดการระบบและควบคุมโหนดเซนเซอร์ของสถานศึกษา:

  • Node Management: เพิ่ม แก้ไข ย้ายพิกัด X-Y ของโหนด และปิดใช้งานโหนดชั่วคราว
  • System Thresholds: ปรับระดับเพดานเตือนภัยค่าฝุ่นละออง PM2.5 ตามนโยบายของโรงเรียน
  • LINE Notify Token: ตั้งค่ากุญแจ API Token ของแอปพลิเคชัน LINE เพื่อเปลี่ยนกลุ่มในการแจ้งเตือนทันที

Work Flow

ขั้นตอนการประมวลผลข้อมูล

การเดินทางของข้อมูลตั้งแต่ชั้นเซนเซอร์ผ่านระบบ AI ไปจนถึงการแจ้งเตือนผู้ใช้งาน

1

ตรวจวัดฝุ่นและสภาพแวดล้อม

โหนดตรวจวัดตามจุดต่างๆ อ่านข้อมูล PM2.5, อุณหภูมิ, ความชื้นสะสม ส่งผ่าน Wi-Fi/LoRa ด้วยโปรโตคอล MQTT ในทุกๆ 10 วินาที

2

ตรวจสอบความเชื่อถือของข้อมูล (DCS)

ระบบ Backend คำนวณหาค่าเบี่ยงเบนสะสมความชื้นสัมพัทธ์ หากมีความชื้นสูงเกินเกณฑ์ ระบบจะลดค่า Data Confidence เพื่อให้แดชบอร์ดระวังค่าดัชนีคลาดเคลื่อน

3

พยากรณ์ฝุ่นด้วย AI Engine

ข้อมูลสภาพแวดล้อมถูกป้อนให้กับโมเดล XGBoost/LightGBM บนคลาวด์/หลังบ้าน ประมวลผลทำนายแนวโน้มฝุ่นในระยะ 1, 3 และ 6 ชั่วโมงข้างหน้า

4

อัปเดตผู้ใช้และส่งการเตือนด่วน

หน้า Dashboard อัปเดตทันที หากระดับฝุ่นเกินขีดจำกัด (เช่น เกณฑ์มาตรฐานที่กำหนด) ระบบจะทำการดึง API ส่งภาพพยากรณ์และข้อความเข้ากลุ่ม LINE
